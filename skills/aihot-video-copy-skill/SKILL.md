# AI 热点短视频文案流水线

## 执行速查

1. **普通请求**：用户只要 AI 热点文案时，直接走默认主线，不追问账号方向。
2. **定制请求**：用户明确要“按账号方向/定位/赛道”来做时，先生成本次账号方向配置，再回到主线。
3. **所有输出**：每条都必须有 `口播稿` 和 `字幕版`。字幕版一句一行，方便复制到剪辑软件。
4. **硬边界**：不编造事实；不做真人出镜；不做录屏实操；不暴露底层内容方法工具名。

## 默认参数

| 项目 | 默认值 |
|---|---|
| 热点来源 | AI HOT 精选条目 |
| 时间窗口 | 最近 24-72 小时，按热点密度调整 |
| 批量数量 | 5 条可用选题 |
| 发布平台 | 抖音、视频号 |
| 视频形态 | 数字人口播 + AI 画面 |
| 视频时长 | 70-85 秒 |
| 默认方向 | 观点、趋势判断、反常识、争议讨论、故事化解读 |
| 禁止方向 | 真人实拍、录屏实操、逐步点击教程、纯新闻复述 |

## 依赖能力

- `aihot`：获取 AI HOT 精选、分类、关键词结果或日报。
- 内容诊断：判断选题、形式、表达和观众价值是否成立。
- 开头优化：优化 5 秒 Hook。
- 标题公式：生成适合短视频的标题候选。
- AI 写作特征检测：定位模板感和 AI 腔；本流程负责后续保真改写。

如果某项能力缺失，只执行仍可完成的部分，并用功能名说明缺失。不要在用户可见输出里写底层工具名或内部前缀。

## 流程

### 0. 判断是否进入账号定制

默认不进入。只有用户明确表达以下意图时才进入：

- 我想要定制化匹配我的账号内容方向
- 按我的账号定位来生成
- 我的账号是面向普通人的 AI 科普号
- 我主要做 AI 热点资讯
- 我的账号做 AI 工具轻教程，不做录屏
- 这批文案要贴合我的账号赛道

如果用户想定制但没说账号方向，只追问一次：

```text
为了让这批文案贴合你的账号，我需要先知道你的账号主要讲什么方向。

你用一句话说就行，比如：面向普通人的 AI 科普、AI 热点资讯、AI 工具轻教程、商业机会解读、工具测评。

这个方向会影响我选哪些热点、用什么角度写，以及字幕节奏怎么切。
```

如果意图有点模糊，用普通中文做短确认，不写英文术语或内部概念：

```text
我理解你是想让这批文案更贴合“AI 工具教程号”的方向。

这会影响我选哪些热点、用什么角度写，以及字幕怎么切。

如果按账号方向做，我会优先选能讲方法和使用场景的热点。
如果不需要定制，我就按默认热点文案流程走。
```

定制只影响本次生产，不保存账号配置，不建立账号库，不让用户填长表。

账号方向配置包含：

- 账号方向
- 目标观众
- 内容风格
- 禁止方向
- 选题筛选逻辑

### 1. 获取热点

使用 `aihot` 获取实时资料，不凭记忆写热点。默认用精选条目；只有用户明确要“全部”“日报”“某分类”“某关键词”时才切换。

每条热点保留：

- 原始标题
- 来源名称
- 原文链接
- 发布时间
- 分类
- 摘要或关键事实

用户可见结果不要暴露 API 参数、游标、限流或调用细节。

### 2. 筛选选题

保留至少满足 3 条的热点：

- 有变化、冲突、趋势或反常识点
- 非专业观众能理解影响
- 有来源和事实，能避免编造
- 能用 AI 画面表达
- 不依赖真人、录屏或软件逐步操作
- 能形成观点，而不只是新闻复述

剔除或降级：

- 没有可追溯来源
- 只是技术更新日志
- 必须真人演示或录屏操作
- 需要编造数据、日期、公司动作或产品能力

账号定制时，再按账号方向筛选：

| 账号方向 | 优先保留 |
|---|---|
| 普通人科普 | 能翻译成普通人影响、低术语的话题 |
| 热点资讯 | 新鲜、重要、有来源、适合当下解释的话题 |
| 轻教程方法 | 能变成使用场景、方法框架或 3-5 个轻步骤的话题 |
| 商业机会 | 改变成本、流程、机会或风险的话题 |
| 工具测评 | 有明确功能变化，能判断适合谁的话题 |

### 3. 生成切入角度

每条入选热点都输出：

- 表面事实：发生了什么
- 核心角度：为什么值得讲
- 目标观众：谁该看
- 观看悬念：为什么要继续看
- 内容承诺：看完能明白什么

账号定制时按方向改角度：

- 普通人科普：这和普通人有什么关系
- 热点资讯：发生了什么，为什么现在重要
- 轻教程方法：可以怎么用，但不做录屏步骤
- 商业机会：成本、流程、机会或风险变了什么
- 工具测评：适合谁，不适合谁，值不值得试

### 4. 生成口播稿

每条生成 70-85 秒中文口播稿，约 360-520 个中文字符，以真实口播节奏为准。

默认结构：

1. `0-5 秒`：Hook，独立成立，不依赖标题或封面。
2. `5-20 秒`：只交代可追溯事实。
3. `20-55 秒`：解释冲突、趋势或反常识点。
4. `55-75 秒`：说明对目标观众的影响。
5. `75-85 秒`：明确判断，不用空泛鸡汤收尾。

账号定制时使用对应骨架：

| 账号方向 | 文案骨架 |
|---|---|
| 普通人科普 | Hook → 事实 → 白话解释 → 普通人影响 → 判断 |
| 热点资讯 | Hook → 谁做了什么 → 关键变化 → 短期影响 → 为什么值得关注 |
| 轻教程方法 | Hook → 使用场景 → 3 个轻步骤/思路 → 适合谁/不适合谁 → 下一步 |
| 商业机会 | Hook → 事实 → 成本/流程变化 → 谁能借势 → 不要误判什么 |
| 工具测评 | Hook → 变了什么 → 谁该关注 → 谁不用急 → 值不值得试 |

写作要求：

- 像数字人能直接念出来的话。
- 句子短，少书面腔。
- 一个句子最多放一个密集术语。
- 不编造事实、数据、引语或产品能力。
- 教程只做轻方法，不写点击路径、界面操作或录屏脚本。

### 5. 诊断和优化

按顺序使用内容方法能力：

1. 内容诊断：检查选题、结构、表达和观众价值。
2. 开头优化：优化前 5 秒 Hook。
3. 标题公式：生成标题候选，适配抖音/视频号。
4. AI 写作特征检测：定位模板感和 AI 腔。

检测出问题后，只改具体问题句，保留事实和原意。

### 6. 生成 AI 画面方向

每条输出 5-8 个可用于 AI 视频或图片生成的画面节拍。

要求：

- 跟口播时间线一致
- 不依赖真实产品界面，除非来源里有
- 优先用概念画面、信息图、字幕卡、场景模拟
- 适合批量生成

账号定制时画面方向：

- 普通人科普：类比画面、概念可视化
- 热点资讯：新闻感、时间线、关键词字幕
- 轻教程方法：步骤卡片、使用场景、简单流程图
- 商业机会：流程图、成本变化、机会地图
- 工具测评：对比卡片、适用/不适用人群

### 7. 生成字幕版

每条口播稿后必须输出字幕版。字幕版用于复制到剪辑软件，不能是整段。

规则：

- 一行一句，或一行一个自然停顿
- 每行通常 8-18 个中文字
- 单行不超过 24 个中文字，专有名词除外
- 按口播节奏切，不按书面段落切
- 一行只表达一个清楚意思
- 不用项目符号
- 和口播稿事实一致

示例：

```text
这次 AI 更新
真正重要的不是功能变多了
而是它开始接近真实工作流

以前你要自己找资料
自己整理表格
自己复制到不同软件里
```

### 8. 质量检查

每条输出前检查：

- 有来源链接
- 事实能追溯到热点资料
- 口播稿适合 70-85 秒
- Hook 不依赖标题或封面
- 不需要真人、实拍、录屏或软件操作画面
- 至少 3 个标题候选
- 已检查并处理 AI 写作特征
- 有一句一行字幕版，且和口播稿一致

账号定制时追加检查：

- 说明为什么适合当前账号方向
- 文案骨架符合账号方向
- Hook 和标题符合目标观众理解门槛
- 画面方向符合账号内容形态
- 教程类不依赖录屏
- 商业类不给无依据赚钱承诺
- 科普类不堆术语
- 资讯类有来源和判断
- 测评类不假装实测

不达标则改一次；仍不达标就跳过并说明原因。

## 输出格式

默认 Markdown。

```markdown
# AI 热点短视频文案批次：{time_window}

## 批次摘要
| 字段 | 内容 |
|---|---|
| 热点来源 | AI HOT |
| 时间窗口 | {window} |
| 入选数量 | {n} |
| 平台 | 抖音 / 视频号 |
| 视频形态 | 数字人口播 + AI 画面 |
```

账号定制时增加：

```markdown
## 本次账号定制配置

| 字段 | 内容 |
|---|---|
| 账号方向 | {account_direction} |

| 内容风格 | {style} |
| 禁止方向 | {avoid} |
| 选题筛选逻辑 | {selection_logic} |
```

每条卡片：

```markdown
## 选题 {i}：{content_title}

- 热点来源：{source}
- 原始标题：{original_title}
- 原文链接：{url}
- 适配判断：适合 / 谨慎 / 不适合
- 推荐角度：{angle}
- 目标观众：{audience}
- 5 秒 Hook：{hook}

### 口播稿（预计 {duration} 秒）
{script}

### 字幕版
{subtitle_lines}

### AI 画面方向
1. {visual_beat}
2. {visual_beat}
3. {visual_beat}

2. {title}
3. {title}

### AI 味检查与处理

```

账号定制时，每条卡片再加：

```markdown
- 账号适配理由：{why this topic fits the account}
- 内容策略：{strategy used}
- 不采用的角度：{excluded angle and reason}
```

## 异常处理

- 热点太少：调整时间窗口、分类或关键词一次；仍不足就说明不足。
- 事实太薄：不猜，不扩写，标为“信息不足”。
- 选题太技术化：尝试转成观众影响；不成立就跳过。
- 用户要求真人出镜或录屏：说明本流程只做数字人口播 + AI 画面，并给兼容写法。
- 批量超过 10 条：建议分批，避免质量下降。

## 完成标准

- 已获取 AI HOT 热点，或说明缺失原因。
- 每条入选内容都有来源、角度、目标观众、Hook、口播稿、字幕版、画面方向、标题候选、AI 写作特征检查。
- 字幕版一句一行，能直接复制到剪辑软件。
- 文案适合数字人口播 + AI 画面。
- 没有编造或扭曲事实。
- 跳过的选题有原因。
- 用户可见文本不暴露底层内容方法工具名或内部前缀。
